TCP (Transmission Control Protocol)

โปรโตคอลควบคุมการส่งข้อมูลแบบมุ่งเน้นการเชื่อมต่อ ทำงานที่ชั้น Transport (Layer 4)

ภาพรวม TCP

TCP เป็นหนึ่งในโปรโตคอลหลักของชุดโปรโตคอล TCP/IP ทำงานร่วมกับ IP เพื่อให้การส่งข้อมูลมีความเชื่อถือได้และเป็นระเบียบ

การทำงานหลัก

  • การเชื่อมต่อ: ใช้ Three‑way handshake (SYN → SYN+ACK → ACK)
  • การปิดการเชื่อมต่อ: ใช้ Four‑way handshake (FIN → ACK → FIN → ACK)
  • ใช้ ACKs และ sequence number เพื่อยืนยันการรับ
  • ควบคุมการไหลด้วย sliding window
  • ควบคุมความแออัดด้วยอัลกอริธึม (slow start, fast retransmit ฯลฯ)
  • ตรวจสอบความถูกต้องด้วย checksum

คุณสมบัติเด่น

Segment Numbering
ติดตามลำดับข้อมูลด้วยการกำหนดหมายเลขเซกเมนต์
Connection Oriented
มีการเชื่อมต่อระหว่างผู้ส่งและผู้รับจนกว่าจะเสร็จสิ้น
Full Duplex
ส่งข้อมูลได้ทั้งสองทางพร้อมกัน
Flow Control
ป้องกันไม่ให้ข้อมูลล้นเครื่องรับ
Error Control
ตรวจสอบและส่งซ้ำหากข้อมูลสูญหายหรือเสียหาย
Congestion Control
จัดการปริมาณข้อมูลในเครือข่ายเพื่อลดการแออัด

ข้อดีของ TCP

  • เชื่อถือได้ ตรวจสอบข้อผิดพลาดและกู้คืนข้อมูลได้
  • ควบคุมการไหลและป้องกันความแออัด
  • ข้อมูลถึงปลายทางตามลำดับ
  • ใช้งานแพร่หลาย และได้รับการมาตรฐานโดย IETF
  • ทำงานคู่กับ IP เพื่อให้บริการเครือข่ายที่เชื่อถือได้

ข้อเสียของ TCP

  • ขนาดใหญ่ อาจไม่เหมาะกับเครือข่ายขนาดเล็ก
  • ลดความเร็วของเครือข่ายบางกรณี
  • ไม่สามารถใช้กับโปรโตคอลอื่นนอกเหนือ TCP/IP ได้
  • ถูกพัฒนามานาน จึงมีข้อจำกัดบางประการ
